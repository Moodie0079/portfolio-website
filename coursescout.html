Project Overview

CourseScout is a course insight platform that helps university students make informed decisions about their course selections. Currently focused on Carleton University with plans to expand to other universities, the platform analyzes years of student conversations on r/CarletonU to extract real student experiences and provide course insights. The modular architecture enables easy adaptation to other universities' course catalogs and subreddit communities.

How It Works

Students type in a course code (e.g., COMP 1005). The system verifies the course exists in the database and retrieves pre-crawled Reddit data. AI processes discussions in the database to extract insights, ratings, and experiences. Insights are delivered with source citations.

Key Capabilities

The system searches through years of Reddit data from the Carleton University subreddit and stores it in a database. When a student searches for a course, the system retrieves the relevant Reddit discussions from the database, sends them to AI for analysis, and displays the processed information including difficulty ratings, professor insights, pros and cons, and student quotes.

System Architecture

Frontend:

Student enters course code. React validates format with input validation and SQL injection prevention. Makes database query. Displays results with error handling and graceful degradation.

Backend API & Database Operations:

Database Query: Receives course code, queries PostgreSQL database using Prisma ORM, retrieves pre-stored Reddit posts and comments for that course.

AI Analysis: Processes up to 1,000 discussions per course query with strict content filtering to prevent course code contamination. Sends to OpenAI GPT-4o-mini and receives structured analysis with difficulty ratings, pros/cons, and student quotes.

Professor Data: Extracts professor names from Reddit discussions, fetches ratings and reviews from RateMyProfessors API, combines with course insights.

Response: Packages AI insights, professor data, and source citations with confidence scoring, returns to frontend.

Data Collection & Crawling System:

Periodically, a crawl will be run to collect data on courses. The crawling process works as follows:

Course Discovery: Scrapes Carleton University's official course catalog, extracts 4,323+ course codes and details, stores in database.

Reddit Crawling: Searches r/CarletonU subreddit for each course code, analyzes up to 500 comments per post, respects Reddit API rate limits with built-in duplicate detection and data validation.

Progress Tracking: Tracks crawling progress in JSON files, can resume interrupted jobs, handles API failures with retry logic, reports completion status.

Database Storage: Stores all collected data in structured PostgreSQL tables, creates relationships between courses, posts, and mentions, enables fast queries.

Challenges & Solutions

Initial Live API Approach:

Problem: Initial version made live Reddit API calls for each course request, taking several minutes per course and hitting rate limits frequently, making the system slow and inefficient.

Solution: Completely redesigned to use a database-driven architecture where data is pre-crawled and stored, significantly reducing response times.

Rate Limiting During Database Crawling:

Problem: Reddit API rate limits during the bulk crawling process for populating the database caused failures and slow data collection.

Solution: Implemented intelligent rate limiting with header tracking, added retry logic with exponential backoff, and built progress tracking to resume interrupted crawling jobs.
