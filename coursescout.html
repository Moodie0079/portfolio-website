CourseScout

AI-powered course insight platform that helps university students make informed decisions about their course selections through Reddit analysis and AI insights. Currently focused on Carleton University with plans to expand to other schools.

Try CourseScout Live View on GitHub

Project Overview

CourseScout is a course insight platform that helps university students make more informed decisions about their course selections. The platform analyzes years of student conversations from the school's reddit and stores information in a database.

How It Works

Students type in a course code and the system queries the PostgreSQL database to retrieve Reddit data. The system searches through a database of pre-crawled Reddit discussions from the university's subreddit. The system retrieves relevant discussions from the database, sends them to AI for analysis, and displays information including difficulty ratings, workload assessments, professor insights, pros and cons, and student quotes with citations.

Performance:

  • First search: 10-20 seconds (full AI analysis)
  • Repeat searches: <100ms (served from cache)

System Architecture

Frontend:

  • Student enters course code
  • React validates format with input validation and SQL injection prevention
  • Makes API request to backend
  • Displays results with error handling and loading states

Backend API & Database Operations:

Database Query: Receives course code, queries PostgreSQL database using Prisma ORM with GIN-indexed arrays for fast course code lookups, retrieves stored Reddit posts and comments.

AI Analysis: Processes up to 1,000 discussions per course query with strict content filtering. Sends to OpenAI GPT-4o-mini and receives structured analysis with difficulty ratings, workload scores, pros/cons, and student quotes.

Professor Data: Extracts professor names from Reddit discussions, fetches ratings from RateMyProfessors API, caches in database for 30 days so information doesn't get outdated, then displays with course insights.

Caching: Stores complete AI analysis results in database as JSON, enabling instant repeat queries and reducing OpenAI costs.

Data Collection & Crawling System:

The crawling process runs periodically to populate the database:

Course Discovery: Loads course codes from University's official course catalog (4,000+ courses).

Reddit Crawling: Searches school's reddit for each course code with multiple format variations (e.g., "COMP 1005", "COMP1005"), fetches complete comment trees, respects Reddit API rate limits with exponential backoff, includes duplicate detection and automatic resume capability.

Course Code Extraction: Analyzes all posts and comments to extract mentioned course codes, stores as arrays in PostgreSQL with GIN indexing for fast searches, enables cross-course discovery.

Database Storage: Stores all collected data in structured PostgreSQL tables with relationships between courses, posts, and comments.

Challenges & Solutions

Initial Live API Approach:

Problem: Initial version made live Reddit API calls for each course request which took several minutes per course and hit rate limits, making the system slow and inefficient.

Solution: Completely redesigned to use a database-driven architecture where data is pre-crawled and stored, reducing response times from minutes to seconds for first-time queries and milliseconds for cached queries.

Rate Limiting During Database Crawling:

Problem: Reddit API rate limits during the bulk crawling process for populating the database caused failures and slow data collection.

Solution: Implemented rate limiting with header tracking, added retry logic with exponential backoff, built progress tracking to resume interrupted crawling jobs, and added error handling for API failures.

Performance Optimization:

Problem: Searching for courses mentioned across thousands of posts was slow with standard database queries.

Solution: Implemented PostgreSQL GIN (Generalized Inverted Index) indexing on course code arrays, enabling fast array containment searches and reducing query times to under 500ms.
