AI Image Classifier

A custom-built convolutional neural network that classifies images of cats and dogs, developed entirely from scratch using TensorFlow and Flask.

Try Live Demo ↗

Project Overview

This image classification web application was developed entirely from scratch using Python, TensorFlow, Flask, and vanilla frontend. It allows users to upload an image of a cat or dog and returns a prediction based on a custom trained convolutional neural network (CNN).

Model Info

This model was trained on a dataset of ~19,000 labeled images of cats and dogs. The model was implemented using tf.keras.Sequential, allowing layers to be stacked sequentially so data flows straight through from the input image to the final prediction. The network was built entirely from scratch without pretrained weights or external model libraries.

Architecture: Three convolutional blocks (Conv2D + MaxPooling2D) followed by a Flatten layer and dense layers with softmax activation for binary classification.

Key Features

  • Custom trained convolutional neural network (CNN) for binary image classification
  • Trained on ~19,000 labeled images of cats and dogs
  • Built using tf.keras.Sequential architecture from scratch without pretrained weights
  • Three convolutional blocks (Conv2D + MaxPooling2D) with dense layers and softmax activation
  • Data augmentation (RandomFlip, RandomRotation, RandomZoom) applied to training set to prevent overfitting
  • EarlyStopping callback that monitors validation loss and restores best weights
  • GPU optimization with dynamic memory allocation using tf.config.experimental.set_memory_growth()
  • Flask backend that accepts image uploads, preprocesses with Pillow, and returns predictions
  • Plain HTML frontend styled with TailwindCSS with JavaScript handling image preview and results display
  • Model saved in .h5 format and loaded by Flask app for inference

Technical Implementation

Backend:

After training, the model was saved in .h5 format and loaded by a Flask app. The backend exposes a GET route for this page and a POST route (/predict) that accepts an image, preprocesses it with Pillow, and returns JSON containing the predicted label and confidence.

Frontend:

The interface is plain HTML styled with TailwindCSS. JavaScript handles image preview, disables/enables the Predict button, shows a spinner during inference, and animates the confidence bar once a result is received.

Problems & Solutions

Overfitting Issues:

Problem: Initial accuracy was poor due to overfitting.

Solution: Introduced data augmentation (RandomFlip, RandomRotation, RandomZoom) applied only to the training set. Also added an EarlyStopping callback that monitors validation loss and restores the best weights when no improvement is seen for 10 epochs.
